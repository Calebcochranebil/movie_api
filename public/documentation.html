MyFlix Movies catalog

Objective

The objective of this application is to develop the server-side infrastructure for a web-based "movies" application, which will give users a comprehensive insight into diverse movies, directors, and genres. Alongside user authentication and profile maintenance capabilities, the application shall empower users to curate a catalog of their preferred movies.

Request URL Response
Endpoint Description Method Query Parameters Request Body Format Example Request Body Response Data Format
/movies Returns a list of all movies in the database. GET None None N/A List of movie objects in JSON format.
/movies/:Title Returns details about a single movie, by title. GET Title (string) None N/A Movie object in JSON format.
/movies/genres/:Name Returns details about a genre, by name. GET Name (string) None N/A Genre object in JSON format.
/users Allows new users to register. POST None User object in JSON format: { "Username": string, "Password": string, "Email": string, "DOB": string } { "Username": "johndoe", "Password": "password123", "Email": "johndoe@example.com", "DOB": "1990-01-01" } User object in JSON format.
/users/:Username Returns data about a user, by username. GET Username (string) None N/A User object in JSON format.
/users/:Username Allows users to update their user information (username, password, email, date of birth). PUT Username (string) User object in JSON format: { "Username": string, "Password": string, "Email": string, "DOB": string } (all fields optional) { "Password": "newpassword123" } User object in JSON format.
/users/:Username/movies/:MovieID Allows users to add a movie to their list of favorites. POST `Username`: string, `MovieID`: string N/A Status code: 201 - Created, 404 - Movie not found, 400 - Bad Request
/users/:Username/movies/:MovieID Allows users to remove a movie from their list of favorites. DELETE `Username`: string, `MovieID`: string N/A Status code: 200 - OK, 404 - Movie not found, 400 - Bad Request
